# Модуль Bypass (Обход блокировок)

Привет! Эта директория и связанные с ней файлы подготовлены специально для тебя. Твоя задача — реализовать обход белых списков (или роутинг) для VPN-пользователей. Мы постарались максимально изолировать этот функционал, чтобы минимизировать Git-конфликты.

## Что уже подготовлено:

1. **База данных (`bypass_rules`)**
   - В `app/db/models/bypass.py` создана SQLAlchemy модель `BypassRule`.
   - Под нее уже сгенерирована Alembic-миграция и таблица в базе. 
   - *Твои действия:* Можешь добавлять нужные тебе колонки (например, `domain`, `routing_tag`) прямо в эту модель и создавать новые миграции (`alembic revision --autogenerate -m "update bypass rules"`).

2. **Бизнес-логика (эта папка)**
   - Скелет класса сервиса находится в `service.py`. Здесь пиши методы, которые будут доставать правила из БД и отправлять команды на VPN-ноды.

3. **Telegram Бот**
   - В `app/bot/handlers/bypass.py` создан базовый роутер и команда `/bypass` для тестов. Этот роутер уже подключен к основному диспетчеру aiogram.
   - *Твои действия:* Добавляй сюда новые хендлеры, FSM стейты или инлайн-клавиатуры для взаимодействия с юзером.

4. **Smart Agent (На стороне VPN-серверов)**
   - В репозитории в папке `smart_agent/api/bypass.py` заготовлен эндпоинт FastAPI. Бэкенд должен отправлять HTTP запросы на этот эндпоинт.
   - В `smart_agent/services/bypass_manager.py` лежит контроллер, который непосредственно управляет нодой (здесь можно вызывать bash скрипты, настраивать iptables или Unbound DNS).
   
## Правила работы
- **Не ломай генерацию ключей:** Старайся не трогать `app/services/vpn_manager.py` или генераторы конфигов AWG/Xray в `smart_agent`, если это не нужно для твоего роутинга. Если нужно — обязательно согласуй.
- **Инфраструктура:** Если для ноды нужны дополнительные системные утилиты (например, `warp-cli`), добавь их в установку `apt-get` внутри `smart_agent/Dockerfile`.
- **Unit of Work:** Для доступа к таблице добавь свой репозиторий в `UnitOfWork` (или обращайся напрямую, если создашь `BypassRepository`).
