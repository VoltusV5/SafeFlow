
# Мониторинг и логирование системы (Sentry + Prometheus + Grafana)

Этот документ определяет архитектуру и целевые метрики системы мониторинга для Telegram-бота и распределенной сети VPN-нод. Наша цель — обеспечить 100% наблюдаемость (Observability) системы на трех уровнях: **код приложения**, **инфраструктура/сеть** и **бизнес-показатели**.

---

## Архитектурная схема мониторинга

```
                     ┌──────────────────┐
                     │      Sentry      │ <─── Ошибки в Python-коде (Бот, API-агент)
                     └──────────────────┘
                              ▲
                              │
┌──────────────────┐  ┌───────────────┐  ┌───────────────────────┐
│  Node Exporter   │  │   FastAPI     │  │   Blackbox Exporter   │
│  (Железо, Сеть)  │  │ (Бизнес/БД)   │  │  (Пинг, SSL, Домены)  │
└──────────────────┘  └───────────────┘  └───────────────────────┘
          │                   │                      │
          └─────────────┬─────┴──────────────────────┘
                        ▼
               ┌─────────────────┐
               │   Prometheus    │ (Сбор и хранение временных рядов)
               └─────────────────┘
                        │
                        ▼
               ┌─────────────────┐
               │     Grafana     │ (Визуализация на едином дашборде)
               └─────────────────┘

```

## 1. Мониторинг ошибок кода: Sentry

Sentry интегрируется на уровне Python-кода в два наших основных компонента: **Telegram-бот (aiogram)** и **VPN Node Agent (FastAPI)**.

### Что отслеживаем:

* **Исключения в хэндлерах бота (`aiogram`):** Ошибки отправки сообщений, сбои при формировании инлайн-клавиатур, падения при парсинге некорректного ввода пользователя.
* **Ошибки работы с базой данных (PostgreSQL):** Истечение таймаута подключения к пулу (`asyncpg`), ошибки транзакций, падения при неконсистентных данных.
* **Сбои интеграции с платежными шлюзами:** Ошибки валидации вебхуков оплаты, таймауты при генерации инвойсов.
* **Ошибки сетевого взаимодействия между Ботом и API-агентами:** Недоступность удаленной VPN-ноды по HTTP, таймауты запросов через `aiohttp`.
* **Ошибки системных вызовов на нодах:** Ошибки вызова консольных утилит (например, сбои при добавлении пира в WireGuard или конфига в Xray через API-агента).

> **Правило алертинга:** Любой критический эксепшен (уровень `Error` и выше) отправляет мгновенный traceback в рабочий чат разработчиков в Telegram.

## 2. Системный и сетевой мониторинг: Prometheus + Grafana

Данные собираются с помощью системных агентов (`Node Exporter` и специализированных экспортеров для WireGuard/Xray) на каждой VPN-ноде и на сервере базы данных.

### А. Метрики операционной системы и железа (Node Exporter)

| Метрика | Что замеряем | Критический порог (Алерт) | Зачем это нужно |
| --- | --- | --- | --- |
| **CPU Utilization** | Загрузка процессора в % | `> 85%` в течение 5 минут | Выявление зависших процессов, бесконечных циклов в коде или злоупотреблений ресурсами. |
| **Memory Usage** | Занятая оперативная память (RAM) | `> 90%` | Предотвращение падения Docker-контейнеров по ошибке Out-Of-Memory (OOM Killer). |
| **Disk Space (Root partition)** | Свободное место на SSD | `> 85%` заполнено | Критично для сервера БД (чтобы не заблокировалась запись) и для нод (где могут копиться системные логи). |

### Б. Сетевые метрики (Критично для VPN)

| Метрика | Что замеряем | Критический порог (Алерт) | Зачем это нужно |
| --- | --- | --- | --- |
| **Network Bandwidth In/Out** | Общая скорость входящего/исходящего трафика в Gbps | `> 70%` от максимальной пропускной способности порта VPS | Сигнал о необходимости масштабирования (закупки новой ноды в этой локации) или признак аномального скачивания торрентов. |
| **Packet Loss** | Процент потерянных сетевых пакетов | `> 2%` потерянных пакетов | Сигнализирует о деградации качества каналов связи у хостинг-провайдера. |
| **Active Tunnel Connections** | Количество активных сессий (клиентов онлайн) | `> 150` активных пиров на одно ядро процессора | Определение лимита физической емкости конкретного сервера. |
| **Abuse Port Activity** | Попытки отправки пакетов через заблокированный порт 25 (SMTP) | Любое значение `> 0` | Сигнализирует о том, что клиент пытается рассылать спам. Требует немедленной автоматической блокировки ключа. |

## 3. Мониторинг доступности и сети: Blackbox Exporter

Работает снаружи системы (независимый опрос сервисов). Заменяет внешние пинговалки.

* **Доступность API-интерфейсов:** Проверка эндпоинта `GET /health` на центральном сервере FastAPI и на всех удаленных VPN-нодах. Код ответа должен быть строго `200 OK`.
* **Срок действия SSL-сертификатов:** Отслеживание количества дней до окончания действия Let's Encrypt сертификатов на наших доменах (например, `nl-1.yourdomain.com`).
* *Алерт:* Осталось `< 14 дней`.


* **DNS Resolution (Резолв доменов):** Проверка того, что доменные имена серверов корректно переводятся в IP-адреса через Cloudflare DNS. Это гарантирует, что клиенты смогут подключиться при динамической смене IP сервера.

## 4. Бизнес- и продуктовые метрики (Custom FastAPI Metrics)

Мы пишем кастомные экспортеры в коде нашего FastAPI-приложения, которые отдают бизнес-метрики из PostgreSQL в Prometheus.

* **Метрики пользователей:**
* `vpn_total_users` — общее количество зарегистрированных пользователей в базе данных.
* `vpn_active_subscribers` — количество пользователей с активной оплаченной подпиской.


* **Метрики ключей:**
* `vpn_keys_generated_total` — общее количество созданных конфигураций.
* `vpn_keys_active_by_node` — количество активных ключей, привязанных к конкретной локации (Нидерланды, Германия и т.д.).


* **Финансовые метрики:**
* `vpn_payments_success_total` — количество успешных транзакций (для отслеживания конверсии оплат).
* `vpn_revenue_rub_total` — накопительный итог выручки для вывода финансовых графиков на дашборд.

## Структура единого дашборда Grafana

Наш главный дашборд должен быть разбит на 4 визуальные зоны:

1. **System Status (Здоровье системы):** Светофоры доступности бота, API-сервера и баз данных (зеленый/красный). Оставшийся срок SSL-сертификатов.
2. **Infrastructure (Серверы):** Графики утилизации CPU/RAM/Диска по каждому серверу с возможностью фильтрации по странам (локациям).
3. **VPN Network (Сеть):** Графики текущей скорости сети (Mbps), объема потребленного трафика за сутки и количества активных пользователей онлайн на каждом сервере.
4. **Sales & Users (Бизнес):** Графики регистраций новых пользователей, динамика покупок подписок и конверсия платежной системы.
