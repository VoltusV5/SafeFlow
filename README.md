# 🛡️ VPN Telegram Bot (SafeFlow Enterprise)

Масштабируемый, отказоустойчивый Telegram-бот и SaaS-платформа для автоматизированного управления, выдачи и мониторинга персональных VPN-туннелей. 

Система спроектирована по принципам **Clean Architecture** и микросервисного подхода: центральный узел (Бэкенд) управляет распределенными по миру VPN-нодами (Smart Agents), обеспечивая бесперебойный доступ пользователей к свободному интернету через безопасные протоколы (AmneziaWG, Xray/VLESS).

---

## 🚀 Ключевые возможности

* 🌍 **Омниканальность:** Управление подписками доступно тремя способами: 
  * Классический Telegram Бот (команды и кнопки).
  * **Telegram Web App (TWA)** — встроенное красивое мини-приложение в Telegram.
  * **Standalone WEB Dashboard** — полноценный личный кабинет в браузере (вход по Telegram OAuth).
* 🔄 **Бесшовный Failover:** Клиенты подключаются к нодам через доменные имена (Cloudflare). В случае блокировки ноды РКН, система переключает A-запись DNS, и у всех пользователей интернет продолжает работать без необходимости перевыпуска ключей.
* 🔐 **Защита от DNS Leaks и Abuse:** На каждой VPN-ноде поднят локальный `Unbound` DNS для предотвращения утечек провайдеру. Весь P2P (торренты) и почтовый спам (Port 25) жестко блокируются.
* 💳 **Продвинутый Биллинг:** Интеграция с High-Risk агрегаторами (Payok, AAIO). Поддержка реферальной системы (+30 дней за друга), промокодов, апгрейдов и полуавтоматических возвратов.
* 📊 **Enterprise Мониторинг:** Интеграция с **Grafana Alloy** (единый агент логов и метрик). Полный контроль над железом нод (Node Exporter), доступностью (Blackbox) и бизнес-метриками FastAPI.
* 👥 **QoS и Шейпинг:** Умное ограничение скорости (через `tc`) при нагрузках на серверах для пользователей базовых тарифов, чтобы премиум-клиенты получали максимум пропускной способности.

## 🛠 Технологический стек

* **Язык:** Python 3.10+ (Строгая типизация, Mypy, полностью `asyncio` код).
* **Архитектура:** Clean Architecture (Repositories, Services, Transport Layers).
* **Методология:** TDD (Test-Driven Development) с использованием `pytest`.
* **Backend Framework:** FastAPI (REST API).
* **Telegram API:** `aiogram 3.x` (поддержка Webhooks в PROD, Long Polling в DEV).
* **Базы данных:** 
  * **PostgreSQL** — Единый источник правды (Single Source of Truth). ORM `SQLAlchemy 2.0` + `Alembic` миграции. Конфиденциальные данные шифруются через `Fernet`.
  * **Redis** — Кэширование, хранение FSM aiogram, брокер фоновых задач и In-Memory троттлинг алертов.
* **Инфраструктура:** Docker, Docker-Compose, GitHub Actions CI/CD.

## 📂 Структура репозитория (Clean Architecture)

```text
vpn-telegram-bot/
├── app/                     # Ядро центрального сервера
│   ├── api/                 # FastAPI роутеры (REST API)
│   ├── bot/                 # Хендлеры aiogram (интерфейс Telegram)
│   ├── core/                # Конфигурация Pydantic и безопасность
│   ├── db/                  # Модели SQLAlchemy, репозитории, миграции
│   ├── services/            # Изолированная бизнес-логика (Billing, VPN Manager)
│   ├── tasks/               # Фоновые процессы (Redis Workers)
│   └── main.py              # Инициализация FastAPI
├── smart_agent/             # Stateless-Агент для установки на VPN-ноды
├── frontend/                # Исходный код внешнего WEB Личного Кабинета
├── twa/                     # Исходный код Telegram Web App
├── docs/                    # Архитектурная документация и схемы MkDocs
├── tests/                   # TDD автотесты (pytest)
├── .github/workflows/       # CI/CD пайплайны сборки
└── docker-compose.yml       # Стек для продакшен деплоя
```

## ⚙️ Установка и Развертывание (DevOps)

Для запуска Центрального Сервера локально:

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-org/vpn-telegram-bot.git
   cd vpn-telegram-bot
   ```

2. **Настройка окружения:**
   Скопируйте `template` файл конфигурации:
   ```bash
   cp .env.example .env
   ```
   Внесите свои данные: токены Telegram, доступы к PostgreSQL и ключи Fernet.

3. **Запуск через Docker-Compose:**
   Вся инфраструктура (PostgreSQL, Redis, Бэкенд) поднимается одной командой:
   ```bash
   docker-compose up -d --build
   ```

4. **Миграции Базы Данных:**
   ```bash
   docker-compose exec app alembic upgrade head
   ```

*VPN-ноды настраиваются отдельно путем развертывания легковесного Docker-контейнера из папки `smart_agent`.*
